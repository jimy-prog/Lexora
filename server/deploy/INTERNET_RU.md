# Lexora в интернете — доступ откуда угодно

Без этого шага сайт открывается только **на твоём компьютере** или **в одной Wi‑Fi сети** по локальному IP.

Чтобы заходить **с телефона через мобильный интернет**, **с другого города** и т.д., нужен **бесплатный Cloudflare Tunnel** (не нужен «белый» IP и открытие портов в роутере).

## Что сделать (один раз)

### 1. Аккаунт Cloudflare

Зарегистрируйся на [cloudflare.com](https://www.cloudflare.com/) (бесплатно).

### 2. Домен

Нужен **свой домен** (например купленный у регистратора), который ты **подключишь к Cloudflare** (NS-записи у регистратора на те, что даст Cloudflare).  
Если домена нет — сначала купи любой дешёвый; без домена постоянный красивый URL через этот способ не настроить.

### 3. Создать туннель

1. Зайди в **Zero Trust**: [one.dash.cloudflare.com](https://one.dash.cloudflare.com/)
2. Слева открой **Networks** — в подменю нажми **Connectors** (в новом интерфейсе туннели создаются здесь; отдельного пункта «Tunnels» в боковом меню может не быть).
3. Найди вариант **Cloudflare Tunnel** (или **Create** / **Add connector**) и создай туннель → имя любое, например `lexora`.
4. На шаге установки коннектора выбери **Docker** — появится **токен** (длинная строка, часто начинается с `eyJ...`). **Скопируй его.**

Если не видишь кнопку: вверху панели попробуй **поиск** по слову `tunnel`.

### 4. Публичный адрес (hostname)

В том же мастере туннеля добавь **Public Hostname**:

| Поле | Значение |
|------|----------|
| Subdomain | например `lexora` (или `app`) |
| Domain | твой домен в Cloudflare |
| Type | **HTTP** |
| URL | **`http://app:8000`** |

Важно: именно **`app:8000`**, не `localhost`. Так туннель достучится до контейнера Lexora внутри Docker.

Сохрани. Через пару минут сайт будет открываться по адресу вида **`https://lexora.твой-домен.com`**.

### 5. Токен в проект

Открой файл **`deploy/.env`** в проекте и добавь **новую строку** (или отредактируй, если уже есть):

```env
CLOUDFLARE_TUNNEL_TOKEN=сюда_вставь_скопированный_токен_одной_строкой
```

Сохрани файл.

### 6. Перезапуск

В папке проекта:

```bash
bash deploy/start.sh
```

Скрипт **сам** увидит токен и поднимет контейнер `cloudflared`. После этого заходи по **https**-адресу из Cloudflare, а не по `127.0.0.1`.

---

## Если что-то не открывается

- В Cloudflare (**Networks** → **Connectors** → твой туннель → настройки маршрута / Public Hostname) проверь, что **URL сервиса = `http://app:8000`**.
- Убедись, что контейнеры запущены: `docker compose -f deploy/docker-compose.yml --env-file deploy/.env ps`
- Логи туннеля: `docker compose -f deploy/docker-compose.yml --env-file deploy/.env logs cloudflared`

---

## Компьютер должен быть включён

Пока Lexora крутится на твоём Mac, **Mac и Docker должны быть запущены**. Для настоящего 24/7 без домашнего ПК позже имеет смысл перенести тот же `deploy/` на VPS в облаке.
